# Security Policy

AutoTalon has local shell and file-system capabilities, so private reporting is
important for issues that could affect workspace integrity, credentials, or
policy enforcement.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub private vulnerability
reporting:

https://github.com/XD319/auto-talon/security/advisories/new

Do not open a public issue for sensitive findings. Public issues are appropriate
only for non-sensitive hardening discussions or documentation improvements.

## Response Targets

- Acknowledge the report within 2 business days.
- Provide initial triage within 7 calendar days.
- Provide a fix, mitigation, or remediation plan within 30 calendar days.

If a report needs more time to investigate responsibly, the advisory thread will
be updated with the current status and next expected checkpoint.
